Ansible模块使用

ansible-server将已经完成密钥对配置的机器加入到/etc/ansible/hosts文件中

[clsn]
192.168.162.129
192.168.162.130

查看机器的负载状态

ansible clsn -a "uptime"

语法：ansible 定义的组/单个ip/域名/all -m 模块名 -a "命令"

ansible语法使用#

ansible “*” -m ping

ansible 192.168.1.* -m ping

ansible “*srvs” -m ping

ansible执行结果说明

ansible-doc 查看模块的使用#

ansible-doc -l #列出模块
ansible-doc -s copy #查看copy模块的参数
绿色：表示没有发生任何改变
红色：执行命令操作出现异常
黄色：执行命令后，对受控主机产生影响，发生了配置改变

ping模块：测试网络连通性

ansible-doc -v ping 查看模块的说明，ansible-doc -s file查看模块的具体信息

使用ansible自带模块执行命令 如果要用 > < | & ' ' 特殊功能的字符，只能使用shell模块

chdir 切换目录

[root@m01 ~]# ansible clsn -m command -a "chdir=/tmp pwd"

creates 文件是否存在，不存在就执行命令

[root@m01 ~]# ansible clsn -m command -a "creates=/etc/hosts date" 172.16.1.31 | SUCCESS | rc=0 >> skipped, since /etc/hosts exists removes 文件是否存在，不存在就不执行命令，

[root@m01 ~]# ansible clsn -m command -a "removes=/etc/hosts date" 172.16.1.31 | SUCCESS | rc=0 >> Fri Oct 20 13:32:40 CST 2017

Ansbile 执行模块原理#

ansible执行的原理？

  1. 加载自己的配置文件 默认/etc/ansible/ansible.cfg
  2. 加载自己对应的模块文件，如command
  3. 通过ansible将模块或命令生成对应的临时py文件，并将该 文件传输至远程服务器 的对应执行用户$HOME/.ansible/tmp/ansible-tmp-数字/XXX.PY文件
  4. 给文件+x执行
  5. 执行并返回结果
  6. 删除临时py文件，sleep 0退出

ansible没有c/s架构，是如何管理被控制端的？

ansible通过ssh协议传输数据。 ansible控制端创建密钥对，将公钥传给被控制端，这样就可以实现传输了。 当然你也可以在配置文件中添加被控制端的ssh用户和密码。
